Privacy Policy

Last updated: July 16, 2026

We collect only the information necessary to respond to inquiries you explicitly send us and to operate artifact verification. Auxtho does not intentionally add site analytics, advertising trackers, or advertising cookies. Hosting and security providers may process technical request data as part of delivery and abuse protection.

An Auxtho verification QR URL contains the report ID, artifact integrity hash, and optional export event ID as query parameters. Opening the URL sends that URL to the site-serving and security infrastructure as part of normal page delivery and prefills the identifiers in the page. The page may request verification-service readiness without sending artifact identifiers to the comparison endpoint. Opening or prefilling the page does not itself start an artifact comparison request or intentionally create an application verification audit record; delivery and security infrastructure may separately log the page and readiness requests.

When you explicitly select Run Verification or submit the manual verification form, accepted inputs are sent to Auxtho for comparison. The submitted report ID, artifact hash, and optional export event ID are used for that request but are not copied into a verification application audit record when one is created.

If you optionally select a PDF for a file-bytes check, the file contents remain in your browser. The browser computes a SHA-256 digest and sends only that digest for comparison; the PDF itself is not uploaded, and the file-bytes digest is not written to the verification audit record.

If a comparison request reaches the instrumented application path and its audit write succeeds, the verification API creates a minimized record containing the outcome, reason code, time, expiry time, and a one-way SHA-256 fingerprint derived from the submitted identifiers. Client-side validation, API validation, rate limiting, network failure, or service unavailability may result in no application audit record. A click or submit action is not proof that an application audit record exists. When created, that record does not contain the submitted report ID, artifact hash, export event ID, selected file digest, file contents, email address, or IP address.

Website, CDN, hosting, API, and security providers may separately process request URLs, IP addresses, user agents, request timing, routing data, and security signals in infrastructure logs. Those logs are not verification application audit records, and their fields and retention depend on the provider and deployment. Where the current pilot verifier creates an application audit record, it is intended to include an expiry timestamp 30 days after creation. Deletion occurs only when the deployment's Firestore TTL policy is configured and active, and must be verified operationally. This is a pilot data-minimization policy, not a regulatory-retention guarantee. Do not submit personal, customer-confidential, or secret identifiers; contact Auxtho for a deletion request or retention question.

Auxtho operates this public pilot site. For questions, deletion requests, or data-related requests, contact hello@auxtho.com. Any signed pilot agreement will identify the applicable contracting and data-controller entity; this page does not infer an unconfirmed legal entity.
